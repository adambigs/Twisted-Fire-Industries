1- Welcome

Welcome to Serre HTML5 Landing Pages Pack. We are Graphicfort, and we'd like to thank you for purchasing one of our products, your support is much appreciated.

Serre is HTML5 / CSS3 template based on Bootstrap 4 framework. It allows you to built impressive, better & faster mobile friendly websites. If you are familiar with bootstrap, it will be easy to customize it and if not, you can learn about bootstrap framework from it's official website.

Please consider Themeforest license policies, the purchase you made allows you to use Serre on only one domain. That means a single purchase is needed for each site you will use Serre on. We appreciate everyone following the policy as it allows us to keep supporting and releasing new updates.

2- Installation

Serre HTML5 Landing Pages Pack is HTML5 template which means you have to edit every thing manually, you can use any text editor like Sublime Text program to edit the files, and when you are done, you can upload the final files via any ftp program like FileZilla directly to your main host folder (usually the folder will be www or public_html)

Check this video to learn how to upload your files using filezilla program.

4- Font

Serre uses 2 Google fonts: Poppins (for the heading) and Roboto (for the body text).

To change these fonts follow these steps:

1- Goto Google font website.

2- Pick up any font you would like to use.

3- Replace this line in HTML file with the ones you got from step 2.

<link href="https://fonts.googleapis.com/css?family=Poppins:400,500,600,700,900%7CRoboto:400,500,700&display=swap" rel="stylesheet">

6- Logo

The header height is 72px and by default the logo image height (image height) is 24px, but if you wan to increase the height for example to be 60px, you will need to substract the logo image height from the header height and divide the result by 2.

72 (Header height) - 60 (Logo image height) = 12
12 / 2 = 6

After that open assets/css/style.css file and find these lines:

.gfort-navbar-brand {
    margin: 0 12px;
    font-size: 33px;
    padding: 24px 0;
    font-weight: 600;
    line-height: 24px;
    white-space: nowrap;
    letter-spacing: -1px;
    display: inline-block;
}

Change padding: 24px 0; to padding: 6px 0; and the header height will still at the same height 72px.

7- jQuery

There are several ways to load js files, we will discuss a few of them and you are free to choose what suits you:

7.1- requirejs method:

- It's already included in the downloaded package.

- This method is useful because it will only load specific js files if the page required them.

- EX: If the page have a parallax effect, it will load the parallax effect js file otherwise the parallax effect js file will not be loaded, and that will lead to decrease the server requests and improve the page speed and performance.

7.1.1- Changing js files location:

If for some reason you want to change the js files location, open assets/js/require.js file and at the end of that file you will find these lines:

requirejs.config({
    paths: {
        bootstrap: 'plugins/bootstrap/js/bootstrap.bundle.min',
        gfortpace: 'plugins/gfort-pace/gfortpace.min',
        animsition: 'plugins/animsition/js/animsition.min',
        fitVids: 'plugins/fitvids/jquery.fitvids.min',
        perfectScrollbar: 'plugins/perfect-scrollbar/perfect-scrollbar.jquery.min',
        mediaElementPlayer: 'plugins/mediaelementplayer/js/mediaelement-and-player.min',
        owl: 'plugins/owl-carousel/owl.carousel.min',
        gfortInsta: 'plugins/gfort-insta/gfort-insta.min',
        tweetie: 'plugins/tweetie/tweetie.min',
        fancybox: 'plugins/fancybox/js/jquery.fancybox.min',
        isotope: 'plugins/isotope/isotope.pkgd.min',
        validate: 'plugins/validation/jquery.validate.min',
        parallax: 'plugins/parallax/jquery.parallax-1.1.3.min',
        appear: 'plugins/appear/jquery.appear.min',
        countTo: 'plugins/countto/jquery.countTo.min',
        typed: 'plugins/typed/typed.min',
        animatedGrid: 'plugins/gridrotator/jquery.gridrotator.min',
        downCount: 'plugins/downcount/jquery.downCount.min',
        twentyTwenty: 'plugins/twenty-twenty/twentytwenty.min',
        easyPie: 'plugins/easy-pie-chart/jquery.easypiechart.min',
        scripts: 'scripts'
    },
    shim: {
        bootstrap: {
            deps:['jquery']
        },
        scripts: {
            deps:['jquery']
        }
    }
});

Change the paths without adding .js extension.

7.1.2- Adding new js code:

From the previous lines you will need to add unique module ID (the left side) and the path to the jQuery file (the right side).

EX: if you want to add a new js file called xyz.js and you want to use it at sepecific pages with specific elements, the new line will be looks like:

abc: 'js/xyz'

Where abc is the module ID and js/xyz is the path to the jQuery file xyz.js without .js extension.

After that open assets/js/scripts.js file and add your own JS code inside documentOnReady function.

The code will be looks like:

requirejs(['abc'], function () {

    // Your JS code

});

Also to improved it, you can make a condition, for example if you want to load the js file when the page have a specific element and that element have a class called abcxyz, the code will looks like:

if ($('.abcxyz').length) {
    requirejs(['abc'], function () {

        // Your JS code

    });
}

10- Contact form

Contact forms consist of 2 parts:

10.1- Form validation:

The form validation is driven by jQuery Validation Plugin

For more details you can check the the documentation and demos.

NOTE: If you want to add a sepecific error message like the download packadge, you have to assign a name for the field you want to validate and add the message in assets/js/scripts.js file.

HTML code:

<input class="gfort-form-control" type="text" name="form_name">

JS code:

$(form).validate({
    rules: {
        form_name: 'required'
    },
    messages: {
        form_name: 'Your name is required.'
    }
});

But if you want to make the validation in general meassage (This field is required) all you have to do is adding reuired attribute to the field you want ot validate.

<input class="gfort-form-control" type="text" name="form_name" reuired>

if for some reason you want to disable the jQuery form validation, open assets/js/scripts.js file and find/remove these lines:

// Validation
if ($('.gfort-form-block-validation').length) {
    requirejs(['validate'], function () {
        GRAPHICFORT.formValidationfn.init();
    });
}

10.2- Form email address:

To add your email address (you will receive the emails to this email address, when the visitor send you an email from the website), open assets/php/contact/contact.php file and find this line:

$email_address = 'example@example.com';

Change example@example.com to your email address without removing the single quotes.

For SMTP Server:

If you are using SMTP server then you will not get any emails even if you add your email address, due to the server configuration, and to solve this problem:

1- You have to create a gmail email and don't activate the 2-Step Verification.

2- open assets/php/contact/contact.php file and find these lines:

/* SMTP email username and password
========================================================================== */
$SMTP_Server   = 'false';
$SMTP_Username = 'example@gmail.com';
$SMTP_Password = 'example@gmail.com password';

- Change false to true
- Change example@gmail.com to your gmail email address.
- Change example@gmail.com password to your gmail email address password.

PS: you will not receive the messages to example@gmail.com but you will receive the messages to example@example.com.

10.3- Google reCAPTCHAV3

Require a localhost or live server to work.

To add Google reCAPTCHA to your website, you will need to know your reCAPTCHA keys: site key and secret key, and to get these kesys follow these steps:

1- Goto Google reCAPTCHA website.

2- If you don't have have any sites registered to use the Google reCAPTCHA API:

2.1 Click on the plus sign (Top right hand side)

2.2 Fill up the form with reCAPTCHA v3 option and click on submit button.

PS: if you will work on localhost you will need to add localhost value in the Domains field.

2.3 After that you will be redirect to a page which contains the Google reCAPTCHA Site key and Secret key.

2.4 Copy site key value and open assets/js/scripts.js file and find this line:

reCaptchaSitekey = 'reCaptcha sitekey',

Change reCaptcha sitekey with the copied value.

Copy secret key value and open assets/php/contact/contact.php file and find this line:

$secretKey = 'reCaptcha SecretKey';

Change reCaptcha SecretKey with the copied value.

11- Counter

For live example check this.

You can copy the HTML code start with:

To change the counter value and the speed change these values in the HTML code:

<span
    data-gfort-count-from="0"
    data-gfort-count-to="900"
    data-gfort-count-speed="1000"
    data-gfort-count-interval="50"
    data-gfort-count-decimals="0">
    900
</span>

Where:

data-gfort-count-from The number to start counting from.
data-gfort-count-to The number to stop counting at.
data-gfort-count-speed The number of milliseconds it should take to finish counting.
data-gfort-count-interval The number of milliseconds to wait between refreshing the counter.
data-gfort-count-decimals The number of decimal places to show when using the default formatter.
900 for the mobile version, (the counter is disabled in mobile for performance and this value will be displayed in the mobile).

13- Google maps

Require a localhost or live server to work.

For live example check this.

To add Google maps to your website, you will need to know the latitude and longitude to the location you want to display the maps for, and also an API key to make the map work.

13.1- Longitude and Latitude

To get the latitude and longitude follow these steps:

1- Goto google maps website.

2- Click on Show your location button from the bottom right corner.

3- The browser URL will change and will be something like https://www.google.com/maps/@48.8628277,2.3523733,15z?hl=en.

The numbers 48.8628277 will be the latitude and the numbers 2.3523733 will be the longitude.

13.2- API Key

To create an API key follow these steps:

  1. Go to the Google Cloud Platform Console.
  2. Click the project drop-down and select or create the project for which you want to add an API key.
  3. Click the menu button and select APIs & Services > Credentials.
  4. On the Credentials page, click Create credentials > API key.
    The API key created dialog displays your newly created API key.
  5. Click Close.
    The new API key is listed on the Credentials page under API keys.

After you get the API key, open assets/js/scripts.js file and find this line:

gmapAPIKey = 'Google map API key',

Change Google map API key with your API key.

  2. Goto your lists and if you don't have one, you can create a new list and fill all the fields, then return back to your lists.
  3. Click on the small arrow behind the Stats word.
  4. Choose Signup forms.
  5. Select Embedded forms.
  6. Scroll down till you reach a text box labeled by Copy/paste onto your site.
  7. Find the form action URL and copy it.

To add the forum URL open assets/js/scripts.js file and find this line:

mailchimpFormURL = 'Mailchimp form URL',

Change Mailchimp form URL with your form action URL from step 7.

Twitter feed have 2 parts:

23.1- PHP

To authorize your twitter feed you will need 4 keys:

  1. Consumer Key.
  2. Consumer Secret.
  3. Access Token.
  4. Access Token Secret.

To get these values follow these steps:

  1. Log into twitter apps website.
  2. click on Create an App button.
  3. Fill all the required fields and click Create button for review (The review process could take 2 days to be approved and it's done by Twitter team).
  4. After the app is approved, you will find your app on this page.
  5. Click on Details button.
  6. Click on Keys and tokens to get your 4 keys.
  7. open assets/js/plugins/tweetie/api/config.php file.
// Consumer Key
define('CONSUMER_KEY', 'Twitter API key');
define('CONSUMER_SECRET', 'API secret key');

// User Access Token
define('ACCESS_TOKEN', 'Twitter access token');
define('ACCESS_SECRET', 'Twitter access token secret');

Change

Twitter API key with your Consumer API key.
API secret key with your API secret key.
Twitter access token with your Access token.
Twitter access token secret with your Access token secret.

24.2- Slider attributes

  • data-gfort-owl-slider-items-margin="36" the space between the slider items and each other (Must be positive integer number).
  • data-gfort-owl-slider-loop="yes" yes or no (yes to make the slider loop after it reached the end item).
  • data-gfort-owl-slider-arrows="yes" yes or no (yes to display the slider arrows). There are 3 layouts for the arrows:
    • Layout 1: by adding data-gfort-owl-slider-arrows-layout="layout-1" attribute with the ability to set the arrows location with the attributes:
      data-gfort-owl-slider-arrows-position="right-bottom",
      data-gfort-owl-slider-arrows-position="left-bottom",
      data-gfort-owl-slider-arrows-position="right-top"
      or
      data-gfort-owl-slider-arrows-position="left-top".
    • Layout 2: By adding data-gfort-owl-slider-arrows-layout="layout-2".
    • Layout 3: by adding data-gfort-owl-slider-arrows-layout="layout-3".
  • data-gfort-owl-slider-dots="yes" yes or no (yes to display the slider dots).
  • data-gfort-owl-slider-autoplay="yes" yes or no (yes to make the slider autoplayable).
  • data-gfort-page-direction="ltr" ltr for LTR language or rtl for RTL language.
  • data-gfort-owl-slider-items-xl="1" number of items for desktop >= 1200px (Must be positive integer number).
  • data-gfort-owl-slider-items-lg="1" number of items for tablet landscape >= 992px (Must be positive integer number).
  • data-gfort-owl-slider-items-md="1" number of items for tablet portrait >= 768px (Must be positive integer number).
  • data-gfort-owl-slider-items-sm="1" number of items for mobile landscape >= 576px (Must be positive integer number).
  • data-gfort-owl-slider-items-xs="1" number of items for mobile portrait < 576px (Must be positive integer number).
  • data-gfort-owl-slider-animateIn="fadeIn" the slider items animation (You can pick any animation from this page).
  • data-gfort-owl-slider-animateOut="fadeOut" the slider items animation (You can pick any animation from this page).
  • data-gfort-owl-slider-mouseDrag="yes" yes or no (yes to enable the mouse drag on desktop devices).
  • data-gfort-owl-slider-touchDrag="yes" yes or no (yes to enable the touch drag on mobile devices).
  • data-gfort-owl-slider-thumbnail="yes" yes or no (yes to convert the slider dots to thumbnail).

These attribute should be added to the gfort-block-wrapper gfort-owl-slider owl-carousel classes.

<!-- Slider block -->

    data-gfort-owl-slider-items-sm="1"
    data-gfort-owl-slider-items-xs="1"
    data-gfort-owl-slider-items-margin="0"
    data-gfort-owl-slider-loop="yes"
    data-gfort-owl-slider-arrows="yes"
    data-gfort-owl-slider-autoplay="yes"
